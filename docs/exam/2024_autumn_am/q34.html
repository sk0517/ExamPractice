テクノロジ系 ネットワーク（ICMP） ICMPIPv4エラー報告制御メッセージ

問題文

IPv4ネットワークにおけるICMPの機能として，適切なものはどれか。

ア　MACアドレスだけが分かっているときにIPアドレスの解決を可能にする。
イ　グローバルIPアドレスとプライベートIPアドレスとを相互に変換する。
ウ　送信元ホストへ，IPパケットの送信エラー報告などの制御メッセージを通知する。
エ　ネットワーク内のIPアドレスを一元管理し，クライアントに動的に割り当てる。

正解

ウ：送信元ホストへ，IPパケットの送信エラー報告などの制御メッセージを通知する。

選択肢補足

選択肢 内容 補足
ア MACアドレスからIPアドレスを解決する これはRARP（Reverse ARP）の機能であり、ICMPの機能ではない。なお、IPアドレスからMACアドレスを求めるのはARPである
イ グローバルIPアドレスとプライベートIPアドレスの相互変換 これはNAT（Network Address Translation）やNAPTの機能であり、ICMPの機能ではない
ウ 送信元ホストへIPパケットの送信エラー報告などの制御メッセージを通知 正解。ICMPはIPと同じインターネット層に位置するプロトコルで、IPパケットの送信エラー（宛先到達不能など）の報告や、ping・tracerouteなどのネットワーク診断に使われる制御メッセージを送信元ホストに通知する役割を持つ
エ IPアドレスの一元管理と動的割当て これはDHCP（Dynamic Host Configuration Protocol）の機能であり、ICMPの機能ではない

解き方

  1. 問題文のキーワード・条件を整理する。
  2. IPv4ネットワークにおける「ICMP（Internet Control Message Protocol）」の機能として適切なものを選ぶ。
  3. ICMPの基本的な役割を確認する。
  4. ICMPはIPと同じインターネット層に位置する制御用プロトコルであり、データそのものを運ぶのではなく、IPパケットの送受信に関するエラーや状態を通知するために使われる。
  5. 例えば、ルーターが宛先にパケットを届けられなかった場合に、その旨を示すエラーメッセージ（到達不能など）を送信元ホストに送り返す。
  6. また、ping（エコー要求／エコー応答）やtraceroute（経路追跡）などのネットワーク診断ツールにも利用される。
  7. 各選択肢が説明しているプロトコル・機能を特定する。
  8. ア：MACアドレスからIPアドレスを求める機能 → RARPの説明。
  9. イ：グローバルIPアドレスとプライベートIPアドレスの変換 → NAT/NAPTの説明。
  10. ウ：送信元ホストへの送信エラー報告などの制御メッセージ通知 → ICMPの説明。
  11. エ：IPアドレスの一元管理と動的割当て → DHCPの説明。
  12. ICMPの定義（送信元ホストへのエラー報告・制御メッセージの通知）と、選択肢ウの記述が完全に一致することを確認する。
  13. ア・イ・エはいずれも別のプロトコル（RARP、NAT、DHCP）の機能であり、ICMPの機能ではないことを確認する。
  14. 以上より、ウを正解と判断する。
