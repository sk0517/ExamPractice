テクノロジ系 ハードウェア（デジタル回路設計） FPGAHDL論理合成回路記述言語

問題文

FPGA などに実装するデジタル回路を記述し，論理合成を行うために使用される言語はどれか。

ア　DDL　　イ　HDL　　ウ　UML　　エ　XML

正解

イ：HDL

選択肢補足

選択肢 内容 補足
ア DDL（Data Definition Language） データベースにおいてテーブルやインデックスなどのデータ構造を定義するための言語（CREATE文やALTER文など）であり、デジタル回路の記述とは無関係
イ HDL（Hardware Description Language） 正解。ハードウェア記述言語と呼ばれ、FPGAやASICなどに実装するデジタル回路の構造や動作をテキストで記述し、論理合成ツールによって実際の回路（ゲートレベルの構成）に変換するために用いられる。代表的な言語にVerilog HDLやVHDLがある
ウ UML（Unified Modeling Language） ソフトウェアやシステムの設計・構造を図示するための統一モデリング言語であり、クラス図やシーケンス図などを記述するために使われるが、デジタル回路の論理合成を目的とした言語ではない
エ XML（Extensible Markup Language） データを構造化して記述するための汎用的なマークアップ言語であり、デジタル回路の記述・論理合成とは異なる用途で使用される

解き方

  1. 「FPGAなどに実装するデジタル回路を記述し，論理合成を行う」という記述の意味を確認する。
  2. FPGA（Field-Programmable Gate Array）は、設計者が自由に内部の論理回路構成を書き込める集積回路であり、その回路構成は専用の言語によって記述される。
  3. 「論理合成」とは、記述された回路の動作・構造の記述（ソースコードに相当するもの）を、実際にFPGA上で動作するゲートレベルの回路構成に変換する工程を指す。
  4. 各選択肢の言語の本来の用途を確認する。
  5. DDL：データベースの世界で使われる、テーブル構造などのデータ定義のための言語であり、回路設計とは全く異なる分野の言語。
  6. UML：ソフトウェア工学におけるシステムの設計を可視化・文書化するためのモデリング言語であり、プログラムやシステムの構造を図で表現するために使われるが、回路を論理合成可能な形で記述する言語ではない。
  7. XML：汎用的なデータ記述・交換のためのマークアップ言語であり、設定ファイルやデータのやり取りに広く使われるが、回路の論理合成を目的とした言語ではない。
  8. デジタル回路設計の分野で実際に使われている言語を確認する。
  9. デジタル回路の設計現場では、回路の構造（どのようなゲートや回路ブロックで構成されるか）や動作（信号がどのように振る舞うか）を、プログラミング言語に似た形式で記述するための専用言語が用いられている。
  10. この専用言語群は「HDL（Hardware Description Language：ハードウェア記述言語）」と総称され、代表的なものにVerilog HDLやVHDLがある。
  11. HDLで記述された回路情報は、論理合成ツールによって解析され、FPGAやASIC上で実際に動作する論理回路（ゲートレベルのネットリストなど）に変換される。
  12. 以上より、FPGAなどへのデジタル回路の実装・論理合成のために使用される言語として、選択肢の中で適切なのはイ（HDL）であると判断する。
