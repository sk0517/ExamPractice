ストラテジ系 システム戦略（情報システム戦略・エンタープライズアーキテクチャ） エンタープライズアーキテクチャBRMSRMTRMDRM参照モデル

問題文

エンタープライズアーキテクチャの参照モデルのうち，BRM（Business Reference Model）として提供されるものはどれか。

ア　アプリケーションサービスを機能的な観点から分類・体系化したサービスコンポーネント
イ　サービスコンポーネントを実際に活用するためのプラットフォームやテクノロジの標準仕様
ウ　参照モデルの中で最も業務に近い階層として提供される，業務分類に従った業務体系及びシステム体系と各種業務モデル
エ　組織間で共有される可能性の高い情報について，名称，定義及び各種属性を総体的に記述したモデル

正解

ウ：参照モデルの中で最も業務に近い階層として提供される，業務分類に従った業務体系及びシステム体系と各種業務モデル

選択肢補足

選択肢 内容 補足
ア アプリケーションサービスを機能的観点で分類体系化したサービスコンポーネント これはSRM（Service Component Reference Model：サービスコンポーネント参照モデル）の説明であり、BRMの説明ではない
イ サービスコンポーネントを活用するプラットフォーム・テクノロジの標準仕様 これはTRM（Technical Reference Model：技術参照モデル）の説明であり、BRMの説明ではない
ウ 最も業務に近い階層の業務体系・システム体系と各種業務モデル 正解。BRM（Business Reference Model：業務参照モデル）は、エンタープライズアーキテクチャの参照モデルの中で最も業務に近い階層として位置づけられ、業務分類に従った業務体系及びシステム体系、各種業務モデルを提供する
エ 組織間で共有可能性の高い情報の名称・定義・属性を記述したモデル これはDRM（Data Reference Model：データ参照モデル）の説明であり、BRMの説明ではない

解き方

  1. 問題文のキーワードを整理する。
  2. 「エンタープライズアーキテクチャの参照モデル」のうち「BRM（Business Reference Model）」として提供されるものを選ぶ問題である。
  3. エンタープライズアーキテクチャ（EA）の基本構造を確認する。
  4. EAは、組織の全体最適化の観点から業務とシステムの双方を統一的にモデル化する設計・管理手法であり、ビジネス・データ・アプリケーション・テクノロジの4つの基本体系（アーキテクチャ）から構成される。
  5. 各基本体系に対応する参照モデルを確認する。
  6. ビジネスアーキテクチャ → BRM（Business Reference Model：業務参照モデル）
  7. データアーキテクチャ → DRM（Data Reference Model：データ参照モデル）
  8. アプリケーションアーキテクチャ → SRM（Service Component Reference Model：サービスコンポーネント参照モデル）
  9. テクノロジアーキテクチャ → TRM（Technical Reference Model：技術参照モデル）
  10. 各選択肢を上記の参照モデルと照合する。
  11. アはアプリケーションサービスの機能的分類に関する記述であり、SRMの説明に該当する。
  12. イはサービスコンポーネントを活用するためのプラットフォーム・技術標準に関する記述であり、TRMの説明に該当する。
  13. エは組織間で共有される情報の名称・定義・属性に関する記述であり、DRMの説明に該当する。
  14. BRMの本質的な特徴に着目する。
  15. BRMは参照モデルの中で「最も業務に近い階層」として位置づけられ、業務分類に従った業務体系・システム体系、各種業務モデルを提供するものであり、業務の標準化・共通化（重複業務の排除、共通基盤の導入など）を主な目的とする。
  16. 以上より、「最も業務に近い階層」「業務分類に従った業務体系及びシステム体系と各種業務モデル」という記述がBRMの定義と完全に一致するウを正解と判断する。
