テクノロジ系 開発ツール（バージョン管理システム） Git分散バージョン管理中央リポジトリクローン

問題文

分散開発環境において，各開発者のローカル環境に全履歴を含んだ中央リポジトリの完全な複製をもつことによって，中央リポジトリにアクセスできないときでも履歴の調査や変更の記録を可能にする，バージョン管理ツールはどれか。

ア　Apache Subversion　　イ　CVS
ウ　Git　　エ　RCS

正解

ウ：Git

選択肢補足

選択肢 内容 補足
ア Apache Subversion 中央集中型バージョン管理システム（CVCS）であり、各開発者のローカル環境には作業中のファイルのみが置かれ、履歴情報は中央サーバが一元的に保持するため、サーバにアクセスできないと履歴の調査などができない
イ CVS Subversionと同様に中央集中型のバージョン管理システムであり、履歴情報は中央サーバ側に存在するため、サーバにアクセスできないときには履歴調査ができない
ウ Git 正解。Gitは分散型バージョン管理システム（DVCS）であり、開発者がリポジトリをクローンすると、コミット履歴やブランチ情報を含むリポジトリ全体の完全なコピーがローカル環境に作成される。これにより、中央（リモート）リポジトリにアクセスできない状況でも、ローカルで履歴の調査やコミット（変更の記録）が可能である
エ RCS 1つのファイル単位で版を管理する非常に古典的なバージョン管理ツールであり、複数人による分散開発やネットワーク経由でのリポジトリ共有を前提とした設計ではない

解き方

  1. 問題文のキーワード・条件を整理する。
  2. 「各開発者のローカル環境に全履歴を含んだ中央リポジトリの完全な複製をもつ」「中央リポジトリにアクセスできないときでも履歴の調査や変更の記録ができる」という特徴を持つバージョン管理ツールを選ぶ。
  3. バージョン管理システムの分類（集中型・分散型）を確認する。
  4. 集中型（CVCS）：履歴情報は中央サーバのみが保持し、開発者のローカル環境には基本的に作業中のファイルしか存在しない。Subversion、CVSはこの方式。
  5. 分散型（DVCS）：開発者がリポジトリを取得（クローン）すると、コミット履歴を含むリポジトリ全体がローカルにコピーされる。Gitはこの方式。
  6. 各選択肢のツールの分類を確認する。
  7. Apache Subversion：集中型バージョン管理システムの代表例。
  8. CVS：Subversionより古い集中型バージョン管理システム。
  9. Git：分散型バージョン管理システムの代表例。
  10. RCS：単一ファイル単位の版管理を行う古典的なツールで、ネットワーク越しの分散開発を前提としていない。
  11. 「全履歴を含んだ完全な複製をローカルに持つ」という記述と各ツールの仕組みを照合する。
  12. 集中型のSubversion・CVSでは、ローカルには通常、作業中のファイル（最新版）のみがあり、過去の履歴はサーバ側にしかないため、サーバにアクセスできないと履歴調査ができない。
  13. Gitでは「git clone」によってリポジトリの完全なコピー（全コミット履歴を含む）がローカルに作成されるため、オフラインでも履歴の確認やコミット（変更の記録）が可能である。
  14. 「中央リポジトリにアクセスできないときでも履歴の調査や変更の記録を可能にする」という条件と、Gitの分散型アーキテクチャの特徴が完全に一致することを確認する。
  15. 以上より、ウ（Git）を正解と判断する。
