テクノロジ系 コンピュータ構成要素 VLIWプロセッサ高速化技法命令セットアーキテクチャ

問題文

プロセッサの高速化技法の一つとして，同時に実行可能な複数の動作を，コンパイルの段階でまとめて一つの複合命令とし，高速化を図る方式はどれか。

ア　CISC　　イ　MIMD　　ウ　RISC　　エ　VLIW

正解

エ：VLIW

選択肢補足

選択肢 内容 補足
ア CISC Complex Instruction Set Computer。1命令で複雑な処理を実行できる命令セット方式であり、コンパイル時に複数動作を1命令にまとめる仕組みではない
イ MIMD Multiple Instruction stream, Multiple Data stream。複数のプロセッサが異なる命令を異なるデータに対して並列実行する並列計算機の分類であり、コンパイル段階での複合命令化とは異なる概念
ウ RISC Reduced Instruction Set Computer。命令セットを単純化・縮小してパイプライン処理などで高速化する方式であり、複数動作を1つの複合命令にまとめる方式ではない
エ VLIW 正解。コンパイル時に依存関係のない複数の命令を1つの長い複合命令にまとめ、同時実行させる高速化方式

解き方

  1. 問題文のキーワードを整理する。
  2. 「同時に実行可能な複数の動作」「コンパイルの段階でまとめて」「一つの複合命令」という3つの特徴がポイントになる。
  3. 各選択肢の方式が、この特徴と合致するか確認する。
  4. CISC：1つの命令自体が複雑な処理を行う命令セット設計の方針であり、「複数動作を1つにまとめる」話ではない。
  5. MIMD：プロセッサ（コア）が複数あり、それぞれ異なる命令・データを並列処理する計算機アーキテクチャの分類であり、コンパイル時の複合命令化とは無関係。
  6. RISC：命令セットをシンプルにして1命令1サイクルに近づけ、パイプライン処理などで高速化する設計思想であり、複合命令化とは方向性が逆。
  7. VLIW：コンパイラが依存関係のない複数の命令をあらかじめ解析し、1つの長い命令（複合命令）にまとめてプロセッサに渡す方式。プロセッサ側は命令間の依存関係を判断する必要がなく、ハードウェアを単純化しつつ高速化できる。
  8. 「コンパイル段階で複数動作を1つの複合命令にまとめる」という記述とVLIWの仕組みが完全に一致することを確認する。
  9. 以上より正解はエ（VLIW）と判断する。
