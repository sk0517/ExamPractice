テクノロジ系 ソフトウェア開発管理技術（開発プロセス・手法・プロセスアセスメント） JIS X 33002プロセスアセスメントISO/IEC 33002プロセス成熟度客観的診断

問題文

JIS X 33002:2017（プロセスアセスメント実施に対する要求事項）の説明として，適切なものはどれか。

ア　組織のプロセスを継続的に改善して品質を高めるための要求事項を規定している。
イ　組織プロセスの品質を客観的に診断するための要求事項を規定している。
ウ　プロジェクトの実施に重要で，かつ，影響を及ぼすプロジェクトマネジメントの概念及びプロセスに関する包括的な手引きを規定している。
エ　明確に定義された用語を使用し，ソフトウェアライフサイクルプロセスの共通枠組みを規定している。

正解

イ：組織プロセスの品質を客観的に診断するための要求事項を規定している。

選択肢補足

選択肢 内容 補足
ア 継続的改善による品質向上の要求事項 これはJIS Q 9001（ISO 9001）などの品質マネジメントシステムに関する規格が対象とする内容であり、JIS X 33002の主旨（アセスメント＝診断の実施要求事項）とは異なる
イ 組織プロセスの品質を客観的に診断するための要求事項 正解。JIS X 33002:2017は、診断対象プロセスのアセスメント結果が客観的で、一貫しており、再現可能であり、かつ代表的であることを確実にするための、アセスメント実施に関する最小限の要求事項を定めた規格である。入力・役割及び責任・アクティビティ・出力という要素を中心に、プロセスアセスメントを公平かつ再現性をもって実施するための枠組みを規定している
ウ プロジェクトマネジメントの概念・プロセスに関する包括的手引き これはPMBOKやJIS Q 21500（ISO 21500）などのプロジェクトマネジメントに関するガイドラインが対象とする内容であり、プロセスアセスメントの実施要求事項を定めるJIS X 33002の説明としては不適切
エ ソフトウェアライフサイクルプロセスの共通枠組み これはJIS X 63001（ISO/IEC 12207）などのソフトウェアライフサイクルプロセス規格が対象とする内容であり、プロセスの「アセスメント（診断）」を扱うJIS X 33002の説明とは異なる

解き方

  1. 問題文のキーワードを整理する。
  2. 「JIS X 33002:2017（プロセスアセスメント実施に対する要求事項）」の説明として適切なものを選ぶ問題である。
  3. JIS X 33002の位置づけを確認する。
  4. JIS X 33002は、国際規格ISO/IEC 33002を基に作成された日本工業規格であり、ソフトウェア開発を主たる対象としていたプロセスアセスメント手法（旧ISO/IEC 15504シリーズ）をより広い範囲に適用すべく再構築されたISO/IEC 33000シリーズの一部を構成する規格である。
  5. 規格本文の趣旨を確認する。
  6. JIS X 33002の規格本文では、「診断対象プロセスのアセスメント結果が、客観的で、一貫していて、再現可能であり、かつ、代表的であることを確実にするアセスメント実施のための最小限の要求事項を定義」している旨が明記されている。
  7. 「プロセスアセスメント」というキーワードの意味を確認する。
  8. プロセスアセスメントとは、組織内のプロセス（ソフトウェア開発プロセスなど）がどの程度有効に機能しているかを客観的に評価・診断する活動であり、その診断結果は改善活動や認証取得などに活用される。
  9. 各選択肢を、関連する別規格・概念と照合する。
  10. アはプロセス改善（品質マネジメントシステム）に関する規格の説明に近い。
  11. ウはプロジェクトマネジメントのガイドライン（PMBOK等）の説明に近い。
  12. エはソフトウェアライフサイクルプロセス規格（ISO/IEC 12207等）の説明に近い。
  13. これらはいずれも「プロセスアセスメント（診断）の実施要求事項」というJIS X 33002の主題とは異なる規格・概念の説明である。
  14. 「品質を客観的に診断する」というキーワードに着目する。
  15. イの記述は、JIS X 33002規格本文が定める「客観的・一貫性・再現可能・代表的なアセスメント結果を確実にするための要求事項」という趣旨と最も合致する。
  16. 以上より、組織プロセスの品質を客観的に診断するための要求事項を規定しているという説明であるイを正解と判断する。
