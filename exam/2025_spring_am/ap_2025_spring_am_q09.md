## 問題文

DMA コントローラーの説明として，適切なものはどれか。

ア　MPU では時間が掛かる積和演算を，高速に行う。
イ　仮想メモリ機能，メモリ保護機能などのメモリ管理機能を提供する。
ウ　動作クロックに合わせてカウントするカウントレジスタをもち，それによって時間の経過を保持する。
エ　メモリと入出力装置，又はメモリとメモリとの間のデータ転送を，MPU を介さずに行う。

## 正解

**エ**：メモリと入出力装置，又はメモリとメモリとの間のデータ転送を，MPU を介さずに行う。

## 選択肢補足

| 選択肢 | 内容 | 補足 |
|:--|:--|:--|
| ア | 積和演算を高速に行う | これはDSP（デジタルシグナルプロセッサ）やFPUなど演算処理に特化したハードウェアの説明であり、DMAコントローラーの役割ではない |
| イ | 仮想メモリ・メモリ保護機能を提供する | これはMMU（メモリ管理ユニット）の説明であり、DMAコントローラーの機能ではない |
| ウ | カウントレジスタで時間経過を保持する | これはタイマー（インターバルタイマーなど）の説明であり、DMAコントローラーの機能ではない |
| **エ** | **MPUを介さずにデータ転送を行う** | **正解。DMA（Direct Memory Access）コントローラーは、CPU（MPU）の介在なしにメモリと周辺機器間、あるいはメモリ間で直接データ転送を行う仕組みであり、CPUの負荷軽減と転送の高速化を実現する** |

## 解き方

1. 「DMA（Direct Memory Access）コントローラー」という名称が示す機能の本質を考える。
   - DMAは直訳すると「直接メモリアクセス」であり、CPU（MPU）を経由せずにメモリへ直接アクセスしてデータを転送する仕組みを指す。
2. 通常のデータ転送方式と比較する。
   - CPUが介在する通常の入出力方式では、データの転送のたびにCPUが命令を実行して転送を制御する必要があり、CPUの処理時間が消費される。
   - DMAコントローラーを用いると、CPUは転送開始の指示を出すだけで、実際のデータ転送（メモリ⇔周辺機器間、またはメモリ⇔メモリ間）はDMAコントローラーが単独で行うため、CPUは転送中に他の処理を並行して実行できる。
3. 他の選択肢が示す機能との違いを整理する。
   - ア（積和演算の高速化）：演算処理装置（DSPなど）の役割であり、データ転送とは無関係。
   - イ（仮想メモリ・メモリ保護）：MMU（メモリ管理ユニット）の役割であり、アドレス変換やアクセス制御を担う機能。
   - ウ（カウントレジスタによる時間経過の保持）：タイマー（インターバルタイマー等）の役割であり、時間計測に関する機能。
4. 「MPUを介さずにデータ転送を行う」という記述が、DMAコントローラーの本質的な定義と完全に一致する**エ**を正解と判断する。
