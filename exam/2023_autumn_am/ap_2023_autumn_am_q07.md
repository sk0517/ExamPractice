## 問題文

JavaScriptのオブジェクトの表記法などを基にして規定したものであって、"名前と値との組みの集まり"と"値の順序付きリスト"の二つの構造に基づいてオブジェクトを表現する、データ記述の仕様はどれか。

ア　DOM　　イ　JSON　　ウ　SOAP　　エ　XML

## 参照画像

（なし）

## 正解

**イ**：JSON

## 選択肢補足

| 選択肢 | 内容 | 補足 |
|:--|:--|:--|
| ア | DOM | Document Object Modelの略で、HTML/XML文書をツリー構造のオブジェクトとして扱うためのAPI仕様であり、JavaScriptオブジェクト表記をベースにしたデータ記述形式ではない |
| **イ** | **JSON** | **正解。JavaScript Object Notationの略で、JavaScriptのオブジェクトリテラル表記をベースに、「名前と値の組みの集まり（オブジェクト）」と「値の順序付きリスト（配列）」の2構造でデータを表現する軽量なデータ交換フォーマット** |
| ウ | SOAP | XMLベースのメッセージ交換プロトコルであり、Webサービス間の通信などに用いられる。JavaScriptオブジェクト表記が基になっているわけではない |
| エ | XML | タグによって階層構造を表現するマークアップ言語であり、JavaScriptオブジェクト表記とは独立した別系統のデータ記述言語である |

## 解き方

1. 問題文のキーワードを整理する。
   - 「JavaScriptのオブジェクトの表記法などを基にして規定」「名前と値との組みの集まり」「値の順序付きリスト」という3点が決定的な手がかりとなる。
2. 「名前と値の組みの集まり」と「値の順序付きリスト」という2つの構造に着目する。
   - これはそれぞれ、プログラミングにおける「オブジェクト（連想配列）」と「配列」に対応する概念であり、JSONの仕様書（json.org）でもこの2つを基本構造として定義している。
3. JavaScriptオブジェクト表記との関係を確認する。
   - JSON（JavaScript Object Notation）は、その名のとおりJavaScriptのオブジェクトリテラル表記法を基にして規定されたテキスト形式のデータ記述仕様であり、現在は特定の言語に依存しない汎用的なデータ交換フォーマットとして広く使われている。
4. 他の選択肢との違いを確認する。
   - DOM：HTML/XML文書をオブジェクトのツリーとして操作するためのAPI（プログラミングインタフェース）であり、データの記述仕様そのものではない。
   - SOAP：XMLをベースにしたメッセージプロトコルであり、JavaScriptのオブジェクト表記とは無関係。
   - XML：タグで構造を表現するマークアップ言語であり、JSONとは別系統の歴史的経緯を持つデータ記述形式。
5. 「JavaScriptオブジェクト表記を基にした」「名前と値の組み」「値の順序付きリスト」という特徴がすべて一致するのはJSONのみである。
6. 以上より、**イ（JSON）**を正解と判断する。
