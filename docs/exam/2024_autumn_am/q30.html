テクノロジ系 データベース（SQL・ウィンドウ関数） SQLROW_NUMBERPARTITION BYORDER BYウィンドウ関数

問題文

"成績"表に対して，SQL文1と同一の結果を得るために，SQL文2の a に入れる字句はどれか。

成績
| 学生番号 | 実施回 | 得点 |
|:--|:--|:--|
| S01 | 1  | 70  |
| S01 | 7  | 80  |
| S02 | 2  | 85  |
| S02 | 5  | 82  |
| S03 | 3  | 83  |
| S03 | 9  | 78  |
| S03 | 12 | 90  |
| S04 | 6  | 100 |

〔SQL文1〕

SELECT R1.学生番号, R1.実施回, R1.得点 FROM 成績 R1
    INNER JOIN
    (SELECT 学生番号, MIN(実施回) AS 初回 FROM 成績
        GROUP BY 学生番号) R2
    ON R1.学生番号 = R2.学生番号
    AND R1.実施回 = R2.初回

〔SQL文2〕

SELECT 学生番号, 実施回, 得点
    FROM (SELECT 学生番号, 実施回, 得点, ROW_NUMBER() OVER ( a ) AS 番号
        FROM 成績) R1
    WHERE R1.番号 = 1

ア　ORDER BY 学生番号, 実施回
イ　PARTITION BY 学生番号 ORDER BY 実施回
ウ　PARTITION BY 学生番号 ORDER BY 得点 ASC
エ　PARTITION BY 学生番号 ORDER BY 得点 DESC

正解

イ：PARTITION BY 学生番号 ORDER BY 実施回

選択肢補足

選択肢 内容 補足
ア ORDER BY 学生番号, 実施回（PARTITION BYなし） PARTITION BYを指定しないため、表全体を一つの集合として連番が振られてしまい、学生ごとに番号がリセットされない。学生番号でソートされた最初の1行（S01の最も実施回が小さい行）しか「番号=1」にならず、各学生の初回レコードを取得できない
イ PARTITION BY 学生番号 ORDER BY 実施回 正解。学生番号ごとにグループ（パーティション）を分け、各グループ内で実施回の昇順に並べ替えて連番を振る。各学生のグループ内で実施回が最も小さい行が「番号=1」となり、SQL文1（各学生の初回の実施回・得点を取得する処理）と同じ結果が得られる
ウ PARTITION BY 学生番号 ORDER BY 得点 ASC 学生ごとにパーティションを分ける点は正しいが、得点の昇順で並べ替えるため「番号=1」になるのは各学生の最低得点の行であり、実施回が最小の行（初回）とは一致しない
エ PARTITION BY 学生番号 ORDER BY 得点 DESC 学生ごとにパーティションを分ける点は正しいが、得点の降順で並べ替えるため「番号=1」になるのは各学生の最高得点の行であり、実施回が最小の行（初回）とは一致しない

解き方

  1. 問題文・SQL文1の意図を整理する。
  2. SQL文1は、サブクエリで各学生番号ごとの最小の実施回（初回）を求め、それを元の成績表と結合することで、各学生の「初回（実施回が最も小さい回）」の実施回・得点を抽出している。
  3. bash_toolでSQL文1を実際に実行し、結果を確認する。
  4. S01：実施回1、得点70／S02：実施回2、得点85／S03：実施回3、得点83／S04：実施回6、得点100、という4行が得られる。
  5. SQL文2の構造を確認する。
  6. ROW_NUMBER()ウィンドウ関数を使い、何らかの基準で連番（番号）を振った上で、番号＝1の行のみを抽出する構成になっている。
  7. ROW_NUMBER() OVER (...) の中で指定する「PARTITION BY」と「ORDER BY」の役割を確認する。
  8. PARTITION BY：番号を振る単位となるグループを指定する（指定した列の値が同じ行同士で連番がリセットされる）。
  9. ORDER BY：パーティション内で番号を振る際の並び順を指定する（先頭が番号1になる）。
  10. SQL文1と同じ結果（各学生ごとの「実施回が最小」の行）を得るための条件を考える。
  11. 学生番号ごとに連番をリセットする必要がある → PARTITION BY 学生番号が必要。
  12. パーティション内で実施回の昇順に並べ、最小の実施回の行を1番にする必要がある → ORDER BY 実施回が必要。
  13. bash_toolで各選択肢のSQLを実際に実行し、結果を比較検証する。
  14. ア（PARTITION BYなし）：表全体で連番が振られ、S01の最小実施回の行のみが抽出され、他の学生のデータが欠落する。
  15. イ（PARTITION BY 学生番号 ORDER BY 実施回）：SQL文1と完全に同じ結果（S01:1,70／S02:2,85／S03:3,83／S04:6,100）が得られる。
  16. ウ（ORDER BY 得点 ASC）：得点が最も低い行が抽出され、SQL文1の結果とは異なる。
  17. エ（ORDER BY 得点 DESC）：得点が最も高い行が抽出され、SQL文1の結果とは異なる。
  18. 実際の実行結果がSQL文1と完全に一致したイを正解と判断する。
