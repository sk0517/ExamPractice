テクノロジ系 セキュリティ（脆弱性・攻撃手法） OSコマンドインジェクションHTTPリクエストヘッダーWebセキュリティログ分析

問題文

Web サーバのログを分析したところ，Web サーバへの攻撃と思われる HTTP リクエストヘッダーが記録されていた。次の HTTP リクエストヘッダーから推測できる，攻撃者が悪用しようとしていた可能性が高い脆弱性はどれか。ここで，HTTP リクエストヘッダー中の "%20" は空白を意味する。

〔HTTP リクエストヘッダーの一部〕

GET /cgi-bin/submit.cgi?user=;cat%20/etc/passwd HTTP/1.1
Accept: */*
Accept-Language: ja
UA-CPU: x86
Accept-Encoding: gzip, deflate
User-Agent: （省略）
Host: test.example.com
Connection: Keep-Alive

ア　HTTP ヘッダーインジェクション（HTTP Response Splitting）
イ　OS コマンドインジェクション
ウ　SQL インジェクション
エ　クロスサイトスクリプティング

正解

イ：OS コマンドインジェクション

選択肢補足

選択肢 内容 補足
ア HTTPヘッダーインジェクション HTTPレスポンスヘッダーに改行コードなどを注入し、レスポンスを分割・偽装する攻撃。本ログにはヘッダー注入を示す改行文字等の痕跡がない
イ OSコマンドインジェクション 正解。パラメータ内の「;」はOSのコマンド区切り文字であり、「cat /etc/passwd」というOSコマンドを実行させ、サーバ上の機密ファイルを読み取ろうとする典型的な攻撃パターンと一致する
ウ SQLインジェクション SQL文の構文（OR、UNION、シングルクォートなど）を悪用してデータベースを不正操作する攻撃。本ログにはSQL構文の痕跡がない
エ クロスサイトスクリプティング <script>タグなどを注入し、被害者のブラウザ上で悪意あるスクリプトを実行させる攻撃。本ログにはスクリプトタグ等の痕跡がない

解き方

  1. リクエストの該当パラメータ部分を確認する。
  2. GET /cgi-bin/submit.cgi?user=;cat%20/etc/passwd HTTP/1.1
  3. パラメータuserの値が ;cat%20/etc/passwd となっている。
  4. "%20"はURLエンコードされた空白文字であることを踏まえてデコードする。
  5. デコード後：user=;cat /etc/passwd
  6. デコードされた文字列の構造を分析する。
  7. 先頭の「;」は、多くのシェル（Unix系OSのシェル）においてコマンドの区切り文字として機能する。
  8. つまり、本来の処理（CGIスクリプトの想定する処理）の後ろに、シェルコマンドとしてcat /etc/passwd（/etc/passwdファイルの中身を表示するコマンド）を割り込ませようとしている。
  9. /etc/passwdはUnix系OSにおいてユーザーアカウント情報が格納された代表的な機密ファイルであり、攻撃の標的としてよく利用される。
  10. この攻撃パターンの性質を考える。
  11. Webアプリケーションがユーザー入力をそのままOSのコマンドとして実行してしまう（あるいはシェル経由で実行してしまう）脆弱性を突き、任意のOSコマンドを実行させようとする攻撃である。
  12. 他の選択肢との違いを確認する。
  13. HTTPヘッダーインジェクション：改行コード（CR/LF）を使ったレスポンスヘッダーの偽装が特徴だが、本ログにはその痕跡がない。
  14. SQLインジェクション：'（シングルクォート）、OR 1=1、UNION SELECTなどのSQL構文の悪用が特徴だが、本ログにはその痕跡がない。
  15. クロスサイトスクリプティング：<script>タグやJavaScriptコードの注入が特徴だが、本ログにはその痕跡がない。
  16. 「;」によるコマンド連結と「cat /etc/passwd」というOSコマンドの組み合わせという決定的な特徴と完全に一致するイ（OSコマンドインジェクション）を正解と判断する。
