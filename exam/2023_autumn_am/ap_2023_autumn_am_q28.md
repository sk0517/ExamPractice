## 問題文

更新可能なビューを作成するSQL文はどれか。ここで、SQL文中に現れる基底表は全て更新可能とする。

ア
```sql
CREATE VIEW 高額商品（商品番号, 商品名, 商品単価）
AS SELECT 商品番号, 商品名, 商品単価 FROM 商品 WHERE 商品単価 > 1000
```

イ
```sql
CREATE VIEW 受注商品（商品番号）
AS SELECT DISTINCT 商品番号 FROM 受注
```

ウ
```sql
CREATE VIEW 商品受注（商品番号, 受注数量）
AS SELECT 商品番号, SUM(受注数量) FROM 受注 GROUP BY 商品番号
```

エ
```sql
CREATE VIEW 商品平均受注数量（平均受注数量）
AS SELECT AVG(受注数量) FROM 受注
```

## 参照画像

（なし）

## 正解

**ア**：CREATE VIEW 高額商品（商品番号, 商品名, 商品単価）AS SELECT 商品番号, 商品名, 商品単価 FROM 商品 WHERE 商品単価 > 1000

## 選択肢補足

| 選択肢 | 内容 | 補足 |
|:--|:--|:--|
| **ア** | **WHERE条件のみの単純な射影・選択** | **正解。集約関数やDISTINCT、GROUP BYを含まず、基底表（商品表）の行と1対1に対応する単純な選択・射影のビューであり、各行を一意に元の表の行へ対応づけられるため更新可能** |
| イ | DISTINCTを使用 | DISTINCTは重複行を除去するため、ビューの1行と基底表の1行が1対1に対応しなくなり、一般に更新不可能なビューとなる |
| ウ | GROUP BYとSUM（集約関数）を使用 | 集約関数（SUM）やGROUP BYによる集計を含むビューは、複数の基底表の行が1つの集計結果行に対応するため、更新操作の対象行を一意に特定できず更新不可能 |
| エ | AVG（集約関数）を使用 | 集約関数AVGによって全行が1つの平均値に集約されるため、基底表の行との1対1対応が失われ更新不可能 |

## 解き方

1. 問題文の条件を整理する。
   - 「更新可能なビューを作成するSQL文」を選ぶ問題であり、基底表（参照元のテーブル）はすべて更新可能という前提がある。
2. ビューが更新可能であるための一般的な条件を確認する。
   - ビューが更新可能であるためには、ビューの各行が基底表の行と1対1で対応している必要がある。
   - 具体的には、ビューの定義に以下のような要素が含まれていると、一般に更新不可能になる：集約関数（SUM、AVG、COUNT、MAX、MINなど）、GROUP BY句、DISTINCT、複数表の結合（一部の例外を除く）、集合演算（UNIONなど）。
3. 各選択肢を、上記の条件に照らして検証する。
   - ア：WHERE句で単純に行を絞り込み、列をそのまま選択しているだけであり、集約関数・DISTINCT・GROUP BYのいずれも含まない。ビューの各行は商品表の各行と1対1で対応するため、更新可能。
   - イ：SELECT DISTINCTを使用しており、同じ商品番号が複数の受注に存在する場合、重複が除去されてビューの行数が基底表（受注表）の行数より少なくなる。1対1対応が崩れるため更新不可能。
   - ウ：GROUP BYとSUM（集約関数）を使用しており、複数の受注レコードが商品番号ごとに1行へ集計される。集計結果の行から元の個々の受注レコードを一意に特定できないため更新不可能。
   - エ：AVG（集約関数）を使用しており、受注表の全行が1つの平均値（1行）に集約される。個々の行との対応関係が完全に失われるため更新不可能。
4. 以上の検証から、集約関数・DISTINCT・GROUP BYのいずれも含まず、基底表の行と1対1に対応する単純な選択・射影のみで構成されているのはアだけであることを確認する。
5. 以上より、更新可能なビューを作成するSQL文として**ア**を正解と判断する。
